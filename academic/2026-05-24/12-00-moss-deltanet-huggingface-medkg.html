MOSS: 自改进 Agent 的源级重写系统 — 技术细节展开

P0 self-improving-agent ★★★★★

来源: arXiv:2605.22794 | GitHub: OpenClaw/MOSS

核心机制:

  • 源级重写: MOSS 不依赖外部训练，而是在生产环境中直接修改 Agent 源代码
  • 失败证据驱动: 通过自动采集的生产失败案例，识别薄弱环节并针对性演进
  • OpenClaw 基准: 四任务基准测试，单周期实现 0.25→0.61 分数跃升（+144%）
  • 可工程化路径: 非概念验证，而是已在真实 Agent 基础设施上验证的工程方法

对 STARK_TOWER 的启示:

  • 可考虑将 Hermes Agent 的工具调用失败纳入自改进证据库
  • 源级重写 vs 行为级学习：MOSS 选择了前者，值得对比评估
  • OpenClaw 基准可作为 STARK_TOWER Agent 评估补充

建议路由: → Jarvis（评估 OpenClaw 基准纳入 + 自改进架构可行性）

Gated DeltaNet-2: 线性注意力的擦写分离 — 架构演进分析

P1 ★★★★

来源: arXiv:2605.22791 | GitHub: NVIDIA/DeltaNet

核心创新:

  • 擦写分离: 将线性注意力中的「擦除」和「写入」操作解耦，而非统一处理
  • 作者团队: Ali Hatamizadeh, Yejin Choi, Jan Kautz — NVIDIA + Allen AI 联合
  • 长序列优化: 针对 Agent 长对话记忆场景，可能显著减少 KV Cache 占用

对 Agent 架构的意义:

  • 线性注意力架构正在快速演进，DeltaNet-2 代表最新一代
  • 擦写分离机制可能更适合 Agent 的「记忆更新」场景
  • 与 LCGuard 形成技术关联：KV Cache 共享安全 + 线性注意力优化

建议路由: → Banner（架构趋势跟踪）

LCGuard: 多 Agent 系统的 KV 共享安全防护 — 安全架构分析

P1 multi-agent ★★★★

来源: arXiv:2605.22786

核心问题: 多 Agent 协作中，KV Cache 共享可能导致潜在通信泄漏

解决方案:

  • Latent Communication Guard: 在 KV 共享层增加防护机制
  • 安全一致性: 确保 Agent 间协作时的信息隔离边界
  • 应用场景: 多 Agent 系统中的敏感数据处理、隔离协作

对 STARK_TOWER 多 Agent 架构的意义:

  • Jarvis + Banner + Pepper 协作可能涉及敏感上下文共享
  • LCGuard 提供了一种「共享但隔离」的架构范式
  • 建议在多 Agent 协作设计时纳入安全边界考量

建议路由: → Jarvis（多 Agent 安全架构研究）
