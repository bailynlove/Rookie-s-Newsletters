PACT：标准化多智能体通信协议，Token 成本减半

来源: arXiv:2606.05304 (滑铁卢大学)

概要: 论文系统分析了五种多智能体通信策略，发现没有单一策略普适最优，但所有有效策略都保留了"行动中心信息"。提出的 PACT 协议将智能体间通信建模为公共状态更新问题，将原始输出压缩为紧凑的动作-状态记录。在 OpenHands 上 resolve rate 提升的同时 Token 减少 10%；在 SWE-agent 上输入 Token 降低 50%。

重要性: 多智能体系统的 Token 成本是实际部署的核心瓶颈。PACT 给出了一个即插即用的通用解决方案，对 Agent 框架开发者有直接参考价值。

评分: ⭐⭐⭐⭐⭐ (9.3/10)

建议路由: → Jarvis（代码智能体评估基准验证）

来源: GitHub: chopratejas/headroom

重要性: 与 PACT 论文从不同维度解决同一个痛点——Token 成本。headroom 更偏通用工具输出压缩，与 PACT 的智能体通信协议互补。
