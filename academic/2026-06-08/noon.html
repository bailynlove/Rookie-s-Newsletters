1. Lean4Agent: 用定理证明器形式化验证 Agent 工作流与轨迹

cs.AIcs.SEcs.LO

来源: arXiv:2606.06523

作者: Ruida Wang, Jerry Huang, Pengcheng Wang, Xuanqing Liu, Luyang Kong, Tong Zhang

核心贡献: 将代理工作流和轨迹建模为 Lean4 中的形式化规约，允许对 agent 行为进行数学验证。这是首次将交互式定理证明器直接应用于 agent 轨迹的语义正确性校验。

局限: Lean4 的验证路径需要人工编写规约，难以完全自动化。

重要性: 随着 agent 系统在生产中自主运作，形式化保证从"可选"变为"必需"。此工作为 agent safety 提供了可落地的验证框架。

评分: ★★★★★ (9.0/10) — P0

建议路由: → Jarvis（评估是否可用于 Hermes agent 轨迹验证）

2. Queen-Bee Agents: 面向受控企业 MCP 编排的 BeeSpec 中心架构

cs.SEcs.AImcp

来源: arXiv:2606.06545

作者: Dutao Zhang, Liaotian

核心贡献: 提出 Queen-Bee 多 agent 架构：Queen 控制面检索能力、规划执行、编译 BeeSpec；Bee 专用 agent 在受限工具访问下执行。59 项企业任务上的 task success rate 0.964，零治理失败。原生集成 MCP 连接器 + 租户级隔离 + 审计执行时治理。

局限:

重要性: 直接回应了 MCP 生态中"如何做企业治理"的核心问题。与我们的 radar watchlist 中 MCP ecosystem & fault taxonomy 直接呼应。

评分: ★★★★★ (9.2/10) — P0

建议路由: → Jarvis（roadmap 评估：MCP 治理层参考架构）

3. SWE-Explore: 编码 Agent 如何探索仓库的基准测试

cs.SEagent-benchmark

来源: arXiv:2606.06535 (cs.SE new submissions 列表)

核心贡献: 首个系统评估编码 agent 仓库探索能力的基准。现有 SWE-bench 类基准主要测量最终修复正确性，而 SWE-Explore 关注 agent 如何在仓库中找到相关信息——这是一个之前被忽视的关键能力维度。

重要性: 与 radar 中"code agent benchmarks"集群直接对齐。探索能力是决定编码 agent 在大型仓库中效用的瓶颈。

评分: ★★★★★ (8.8/10) — P0

建议路由: → Jarvis（评估 SWE-Explore 方法论是否需要纳入 / 影响内部 benchmark 设计）

重要性: 与 Socratic-SWE 互补——前者从轨迹蒸馏技能，后者侧重开放世界持续学习。两者构成了 agent self-evolution 的两个侧面。

8. Declarative Skills for AI Agents: 知识接地工具使用工作流

cs.AIdeclarative-skill

来源: arXiv cs.AI new (2026-06-08)

核心贡献: 提出声明式技能（declarative skills）概念——将工具使用工作流建模为知识图上的声明式规约，而非程序式脚本。提高了技能的可解释性和跨域迁移能力。

重要性: 与 Workflow-to-Skill 互补——一个从工作流压缩，一个从知识图构造。两者的结合可能定义下一代 agent skill 格式。

评分: ★★★★ (6.8/10) — P1
