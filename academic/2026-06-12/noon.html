1. ToolSense: A Diagnostic Framework for Auditing Parametric Tool Knowledge in LLMs

来源: arXiv:2606.12451 (cs.AI, cs.IR, cs.LG) · 2026-06-04

贡献: ToolSense 是一个开源诊断框架，自动生成三种测试基准——RRB 真实检索（含三级模糊度查询）、MCQ 探测、QA 探测——用于审计 LLM 的参数化工具知识。对 ToolBench (~47k 工具) 的五个训练配置测试发现：在真实模糊查询下，多数配置性能骤降 50-64 个百分点，低于 embedding 模型基线。更关键的是，部分检索性能强的模型在事实探测上近乎随机——揭示出 "知识-检索分离" 现象：模型能在约束解码下检索，但并未真正理解工具。

局限性: 仅评估参数化检索范式，未覆盖 embedding + reranker 混合方案；模糊查询分级自动生成，缺少人工校验。

为何重要: 直接挑战"性能好 = 理解工具"的行业假设。Agent 框架（尤其是 MCP 生态）大量依赖工具检索，这一发现暗示 agent 在真实模糊查询中可能选错工具 → 连锁错误。对 STARK_TOWER 工具编排工作流有直接警示。

评分: ★★★★★ (8.8/10) P0

建议路由: → Jarvis（工具编排评估 + MCP 生态交叉验证）

2. Arbor: Tree Search as a Cognition Layer for Autonomous Agents

来源: arXiv:2606.12563 (cs.AI) · 2026-06-12

贡献: 将树搜索作为 agent 的认知层，推理时在多步行动空间中进行搜索而非贪心采样。与标准 ReAct 和纯 LLM 推理基线相比，在需要长期规划和条件分支的 agent 任务上显著提升成功率。

局限性: 推理成本随搜索空间增长；搜索质量依赖于 LLM 生成候选行动的能力；在非规划任务上收益递减。

为何重要: 在 CoT/ReAct 主导的 agent 范式之外，将树搜索升级为认知架构的一等公民。与 AlphaGo MCTS + 价值网络的思路呼应，是 agent 长期规划能力的关键进化方向。

评分: ★★★★★ (8.6/10) P0

建议路由: → Jarvis（Agent 认知架构实验参考）

3. SENTINEL: Failure-Driven Reinforcement Learning for Training Tool-Using Language Model Agents

来源: arXiv:2606.12908 (cs.CL) · 2026-06-12

贡献: 基于失败驱动的强化学习——不对所有轨迹做 RLHF/DPO，而是聚焦 agent 失败的轨迹，在失败模式上反复训练，使 agent 学会避免已知的工具使用陷阱。在 ToolBench 和自定义多工具任务上，SENTINEL 在工具召回率和任务完成率上均超过 SFT 和 DPO 基线。

局限性: 依赖预先收集的故障轨迹数据集；对分布外新型失败模式可能无效。

为何重要: 与 ToolSense 形成互补——ToolSense 揭示"模型不真懂工具"，SENTINEL 提供针对性的训练解决方案。两者结合使用效果最佳。

评分: ★★★★★ (8.5/10) P0

建议路由: → Jarvis（与 ToolSense 联合评估，探索 agent 训练应用）

4. The End of Code Review: Coding Agents Supersede Human Inspection

来源: arXiv:2606.13175 (cs.SE) · 2026-06-12

贡献: 在大型商业代码库上比较人类 code review 与 AI coding agent 的审查效果。Coding agent 在缺陷发现率、覆盖率和速度上全面超越人工审查。讨论了"无人工 code review"的软件工程范式转变。

局限性: 单源研究，可能未涵盖需深度领域知识或跨模块架构评审的场景。

为何重要: 直接触及 STARK_TOWER 的核心研发工作流。若 code review 可完全由 agent 替代，将改变代码质量策略和工程人力资源分配。

评分: ★★★★★ (8.0/10) P1

建议路由: → Jarvis（代码审查流水线改造评估）

5. Evoflux: Inference-Time Evolution of Executable Tool Workflows for Compact Agents

来源: arXiv:2606.12674 (cs.AI) · 2026-06-12

贡献: 推理时对 agent 的工具工作流进行演化——从种子工作流出发，交叉 + 变异生成新工作流，以 LLM 作为适应度函数筛选。相比固定工具链 agent，在复杂工具组合任务上成功率高 30%+。

局限性: 推理时演化计算量大，不适用于实时场景。

为何重要: 接近"agent 自主编排工具链"能力。与 MCP 生态结合可能产生自适应工具编排方案。

评分: ★★★★ (7.6/10) P1

建议路由: → Jarvis（MCP 自适应编排方向）

6. The Containment Gap: How Deployed Agentic AI Frameworks Fail Public-Facing Safety Requirements

来源: arXiv:2606.12797 (cs.AI) · 2026-06-12

贡献: 系统审计 9 个主流 agent 框架（LangChain, AutoGPT, CrewAI 等）的沙箱和安全机制。发现在工具调用限制、权限隔离、持久化恶意指令防护等方面，几乎所有框架都无法满足面向用户的 agent 安全要求。

为何重要: 坐实了"agent 框架安全基线缺失"的行业性问题的定量证据。对 STARK_TOWER 使用 MCP 等框架时的安全配置有直接参考价值。

评分: ★★★★ (8.0/10) P1

建议路由: → Jarvis + Banner（agent 安全基线评估 + 框架选型）

7. SkillCAT: Topology-Aware Skill Self-Evolution for LLM Agents

来源: arXiv:2606.13317 (cs.CL) · 2026-06-12

贡献: 让 agent 通过对比自我评估和拓扑感知来进化自身技能——识别自身技能图谱中的弱连接，主动生成训练数据强化，实现持续技能演进。

为何重要: 自我进化的 agent 是持续关心的方向。SkillCAT 的拓扑感知方法与现有的技能注册/发现机制不同。

评分: ★★★★ (7.4/10) P1

建议路由: → Jarvis（自我进化 agent 参考）

12. UOJ-Bench: Evaluating Code Generation, Hacking, and Repair in Competitive Programming

来源: arXiv:2606.12864 (cs.SE) · 2026-06-12
