P0 — 必须关注

📄 SciConBench: Can AI Agents Synthesize Scientific Conclusions?

来源: arXiv:2606.11337 (cs.AI, cs.CL, cs.CY)

贡献: 构建 SciConBench — 含 9.11K 问题 + 专家撰写结论的大规模直播式科学结论合成基准。引入 SciConHarness 洁净室评估框架。评估 8 个前沿模型，最佳 agent 在洁净环境下 factual F1 仅 0.337。审查消费级 agent（Google AI Overview、OpenEvidence）发现即使答案已知也频繁生成不完整甚至矛盾结论。

为什么重要: 这是目前最严谨的科学 AI agent 合成能力评估。数据泄漏对 paper 结论的"污染"量化已被严格隔离。对 STARK_TOWER 来说，直接回答了"我们的 agent 能否可靠检索后合成结论"这一核心问题。

局限: 仅覆盖健康/医学领域；clean-room 设置可能低估实际部署中 web 交互的价值。

评分: ★★★★★ (9.5/10)

建议路由: → Jarvis（评估 SciConHarness 方法是否可用于 STARK_TOWER 内部评测）

📄 Claw-SWE-Bench & ClawEnvKit & ResearchClawBench

来源: Claw-SWE-Bench (cs.CL) · ClawEnvKit (cs.CL) · ResearchClawBench (cs.CL)

贡献: 今日 OpenClaw 生态出现一批新基准：Claw-SWE-Bench 评测 OpenClaw 风格 agent harness 的编码任务能力；ClawEnvKit 实现自动环境生成；ResearchClawBench 建立端到端自主科学研究的评测标准。三篇论文彼此互补形成完整工具链。

为什么重要: OpenClaw 生态正在从单个项目演变为一个评测体系。Claw 类 agent 架构可能在下一阶段与 MCP 竞争 — 这是 STARK_TOWER 必须跟踪的架构方向。

评分: ★★★★★ (9.0/10)

建议路由: → Jarvis（技术路线评估：Claw vs MCP vs 纯 agent 框架）

📄 A Five-Plane Reference Architecture for Runtime Governance of Production AI Agents

来源: arXiv:2606.11240 (cs.AI, cs.SE, cs.CC, cs.CR)

贡献: 提出面向生产环境 AI agent 运行时治理的五平面参考架构：数据平面、控制平面、推理平面、审计平面、策略平面。通过将治理嵌入 agent runtime 而非外部监控来实现可审计、可回滚的 agent 操作。

为什么重要: MCP enterprise governance（radar watchlist 持续跟踪）从概念阶段进入了架构设计阶段。这篇论文提供了可直接实施的蓝图。

评分: ★★★★★ (8.8/10)

建议路由: → Jarvis（与现有 MCP 治理方案对比评估）

P1 — 值得深入

📄 SkillJuror: Measuring How Agent Skill Organization Changes Runtime Behavior

来源: arXiv:2606.11344 (cs.AI, cs.SE)

贡献: 首次量化研究 agent 技能组织方式（扁平列表 vs 层次化 vs 图谱）对运行时行为的影响。发现技能组织方式显著影响任务完成率、token 消耗和失败恢复模式。

为什么重要: 直接关联 agent-skills 项目（GitHub 52K stars）的架构设计决策。技能爆炸已成为社区共识，但"技能如何组织"一直缺乏实证。这篇论文填补了这一空白。

评分: ★★★★ (8.2/10)

建议路由: → Jarvis（引入技能组织策略评估）

📄 HERO: Hindsight-Enhanced Reflection from Environment Observations for Agentic Self-Distillation

来源: arXiv:2606.11347 (cs.AI)

贡献: 提出基于环境观测的 hindsight 增强反思，使 agent 能从失败轨迹中自蒸馏出更好的行为策略。在多种 agent 任务中展示出显著的自我进化能力。

为什么重要: "Agent self-evolution"（radar watchlist 持续跟踪）从概念走向实验验证。HERO 的方法可以直接用于改善 STARK_TOWER agent 的自我改进循环。

评分: ★★★★ (8.0/10)

建议路由: → Jarvis（评估 HERO 是否可集成至现有 agent 训练/改进流程）

📄 PROJECTMEM: A Local-First, Event-Sourced Memory and Judgment Layer for AI Coding Agents

来源: arXiv:2606.11241 (cs.AI)

贡献: 提出面向 coding agent 的本地优先、事件溯源记忆层。通过结构化的事件流实现可审计的 agent 判断过程，支持跨 session 记忆持久化和上下文压缩。

为什么重要: Agent 记忆系统一直是 radar 关注方向。PROJECTMEM 的本地优先 + 事件溯源思路特别适合客户端/私有部署场景。与"Agent memory systems"雷达条目直接对应。

评分: ★★★★ (7.5/10)

建议路由: → Jarvis（评估记忆架构兼容性）

📄 CRANE: Constrained Reasoning Injection for Code Agents via Nullspace Editing

来源: arXiv:2606.11243 (cs.SE, cs.AI, cs.CL)

贡献: 通过零空间编辑（nullspace editing）向代码 agent 注入约束推理，在不改变核心能力的前提下确保 agent 遵守安全/策略约束。实验显示约束注入后 agent 未出现能力退化。

为什么重要: 与五平面治理架构互补 — CRANE 提供了更细粒度的推理层约束机制。对于需要保证 agent 行为安全的生产场景尤为重要。

评分: ★★★★ (7.3/10)

建议路由: → Jarvis（与五平面架构合并评估）
