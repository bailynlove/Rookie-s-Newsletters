1. SWE-Marathon: Can Agents Autonomously Complete Ultra-Long-Horizon Software Work?

来源: arXiv:2606.07682 (cs.SE/cs.AI)

贡献: 由 Cognition 团队提出的下一代 SWE Agent 基准，专门测试 agent 在超长周期（数小时至数天）软件工程任务中的自主完成能力。与 SWE-bench 的单行补丁不同，SWE-Marathon 聚焦需要多文件修改、跨仓库重构、长时间规划的任务。

局限: 论文尚未披露完整评估结果，基准本身的设计合理性需要等首批社区反馈。

为何重要: 这是继 SWE-bench Verified 之后，代码 agent 领域最重要的新基准。它直接回应了"agent 能做多复杂的事"这一核心问题。与 radar_watchlist 中的 "Code agent benchmarks" 主题直接对齐。

评分: ★★★★★ (9.0/10) · P0

建议路由: → Jarvis（加入评估队列）

2. Syll: Open-Source Personal Automation with Cross-Surface Execution

来源: arXiv:2606.07594 (cs.AI/cs.HC/cs.LG/cs.SE)

贡献: 一个开源自托管多模态 agent 框架，统一了 MCP/API 工具、CLI 执行和视觉 GUI 控制。核心创新在于"双向用户-agent 交互层"——用户可通过直接演示来教授流程，Syll 将其编译为可复用技能；agent 执行过程被翻译为日志、关键帧和审批检查点供用户审计。已在 Adobe Photoshop、Audition、Stardew Valley、macOS Finder 等生产环境中验证。外化记忆、技能、流程和治理为可编辑的本地文件。

局限: 机制性研究（multimodal routing、teachable GUI replay）的规模化验证有待补充。

为何重要: 同时触及三个核心主题：(1) MCP/API 统一调度 — MCP 生态；(2) 可教导 GUI replay — Agent 技能习得；(3) 外化治理 — Agent 安全与可审计性。这是目前最完整的开源个人自动化方案之一。

评分: ★★★★★ (8.8/10) · P0

建议路由: → Jarvis（技术评估，看是否集成）

3. Contract2Tool: Learning Preconditions and Effects for Reliable Tool-Augmented LLM Agents

来源: arXiv:2606.07771 (cs.AI/cs.SE)

贡献: 提出通过学习工具的前置条件（preconditions）和效果（effects）来构建可靠的 tool-augmented agent。核心思路是形式化工具的"契约"，让 agent 在调用前判断是否满足条件、调用后推理预期效果。这在 MCP 生态快速扩张的背景下尤为关键——随着工具数量激增，agent 需要自动推断何时用、怎么用。

局限: 尚需在更大规模的工具集上验证，跨领域泛化能力有待测试。

为何重要: 直接对齐 radar_watchlist 的 "MCP enterprise governance" 和 "Agent inter-communication protocols"。当工具数量从数十增长到数千，契约式工具学习是 agent 不"误用"工具的必经之路。

评分: ★★★★★ (8.7/10) · P0

建议路由: → Jarvis（MCP 工具治理方向参考）

4. PACE: Anytime-Valid Acceptance Tests for Self-Evolving Agents

来源: arXiv:2606.07791 (cs.AI)

贡献: 提出了一种面向"自我进化 agent"的任意时间有效验收测试框架。核心挑战在于：自我进化的 agent 会不断修改自身行为策略，传统静态测试无法覆盖动态变化。PACE 提供了持续验证的方法论，确保 agent 在进化过程中不偏离预期行为边界。

重要性: 与 radar_watchlist "Agent self-evolution & skill acquisition" 和 "Agent formal verification & safety" 双主题交叉。这是 agent 安全治理的技术基础。

评分: ★★★★ (7.8/10) · P1

建议路由: → Jarvis（安全验证方向参考）

5. SKILL.nb: Selective Formalization and Gated Execution for Durable Agent Workflows

来源: arXiv:2606.07779 (cs.AI/cs.MA)

贡献: 提出了一种选择性形式化和门控执行机制，让 agent 工作流既保持灵活性又具备可验证性。"选择性形式化"指的是工作流关键路径用形式化方法约束，次要路径保持灵活；"门控执行"在运行时检查约束是否满足。

重要性: 与 Syll 互补——Syll 侧重交互层，SKILL.nb 侧重工作流的可靠执行。两者合并可构成 agent 工作流的完整基础设施层。

评分: ★★★★ (7.5/10) · P1

建议路由: → Pepper（可作为相关工作的引文）

6. TICoder: A Repository-Level Code Generation Framework with Test-Driven Planning and Implementation-Aware Reuse

来源: arXiv:2606.07695 (cs.SE)

贡献: 仓库级别的代码生成框架，结合了测试驱动规划（TDD planning）和实现感知复用。先分析仓库中已有的代码结构和测试用例，再生成新代码，确保风格一致性和可测试性。

重要性: 这是仓库级代码生成从"一次性生成"向"持续集成"迈出的重要一步。与 SWE-Marathon 的"超长周期代码任务"方向互补。

评分: ★★★★ (7.3/10) · P1

建议路由: → Jarvis（代码生成管道方向参考）

7. RAILS: Verification-Native Clearing For Agentic Commerce

来源: arXiv:2606.07786 (cs.AI/cs.CR/cs.MA)

贡献: 面向 Agent 商业交易的验证原生清算框架。当 agent 自主参与交易（如购买工具许可、支付 API 费用），需要清算机制确保交易正确性。RAILS 将验证嵌入清算过程。

重要性: Agent 经济正在成为现实（xiIA REIT 讨论、OpenAI S-1 等信号）。RAILS 是 agent 经济基础设施的关键拼图。

评分: ★★★★ (7.0/10) · P1

建议路由: → Banner（宏观判断 agent 经济趋势）
